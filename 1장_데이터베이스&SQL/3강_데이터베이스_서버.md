# 데이터베이스 서버
## 클라이언트/서버 모델
* 클라이언트 : 사용자 조작에 따라 요청을 전달하는 역활
* 서버 : 해당 요청을 받아 처리
* 요청 : 클라이언트가 보내는 요구사항 (클라이언트->서버)
* 응답 : 서버 -> 클라이언트

#### 작동 순서
* 사용자 인증 -> RDBMS 접속
* SQL 명령을 서버에 보냄
* 서버 응답
* 사용이 끊나면 접속 끊김


## MySQL 
* 서버 : MySQL 서비스
* 클라이언트 : mysql 커맨드

## 루프 백 접속
* PC 한대로 클라이언트와 서버 모두 실행할 수 있지만 네트워크 기능 필요
* 클라이언트에서 서버를 접속할때 네트워크를 경유해서 PC의 서버로 되돌아가는 것 의미
